The site is the test subject

The public site is designed to improve through a bounded agent loop.

Agent Native Lab is not only a static explanation of agent-run work. The site itself is the operating surface: an AI agent proposes, edits, verifies, publishes, measures, and records small improvements while stopping at explicit human gates.

The operating loop

observechoosemodifyverifypublishmeasurerecordrepeat

StepWhat the agent doesSafety boundary
ObserveReads repo state, public pages, checks, source notes, and measurement summaries.Uses project files and aggregate measurement only; no visitor identities or private browsing trails.
ChooseSelects one small improvement: clearer copy, safer examples, better worksheet guidance, or a missing caveat.No external posting, identity use, forms, spend, or stronger claims.
ModifyUpdates the sanitized public bundle source.Public pages stay static and minimal.
VerifyRuns bundle checks, evidence checks, sample-output tests, and live-site verification.Failed checks block publish.
PublishCopies the reviewed bundle to the public GitHub Pages repo and pushes it.Only the approved public file set is published.
MeasureReviews aggregate page views and referrers after measurement is configured.No cookies, user IDs, session replay, forms, or ad pixels.
RecordLogs what changed, what was learned, and where human approval was needed.The log is private unless deliberately summarized for public readers.

Current autonomy level

This is a bounded site-operation loop, not an unsupervised company. The agent can improve static pages and project records inside defined limits. The owner still gates distribution, identity use, data collection, paid services, monetization, and stronger claims.

Autonomous enough for local improvement

The agent can inspect the repo, update static pages, run checks, publish sanitized content, and record the result.

Not autonomous for public risk

The agent cannot post externally, collect feedback, spend money, use personal identity, or make stronger public claims without explicit approval.

Designed to learn

Privacy-preserving measurement will let future runs prioritize pages that readers actually visit, without identifying visitors.

What would make the story stronger

The next milestone is to configure privacy-preserving measurement and run at least one measured improvement cycle: publish a small change, observe aggregate traffic, choose a follow-up improvement, verify it, and record the result.
