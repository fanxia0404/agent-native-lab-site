# Public review bundle

This directory is the only proposed public-facing review bundle for Agent Native Lab.

## Included pages

- `index.html` — home page and value proposition.
- `evaluate-a-workflow.html` — practical worksheet for scoring whether a workflow is ready for agent operation, human review, or manual clarification.
- `how-we-test.html` — explains how the lab tests agent-run workflows and what readers can learn from the test method.
- `self-improving-loop.html` — explains the bounded AI-agent operating loop that improves the site over time.
- `workflow-framework.html` — gives a reusable checklist for judging whether a business workflow is ready for agent operation.
- `real-world-examples.html` — source-linked candidate evidence and pattern extraction.
- `privacy.html` — explains the measurement/privacy boundary before aggregate measurement is enabled.
- Bundle note — this file.

## Not included in a public release

Internal strategy notes, source records, logs, cost records, human-review records, generated dashboards, validation plans, scorecards, git history, forms, email capture, payment links, domain setup, deployment configuration, visitor identities, cookies, session replay, ad pixels, and user IDs are not part of this public review bundle.

## Review intent

The public reader should immediately understand:

1. **Audience:** solo founders and technical builders evaluating AI-agent workflows.
2. **Value:** a practical way to decide which workflows agents can realistically run.
3. **Evidence level:** hypothesis, private local testing, and source-reported candidate evidence — not proof of market demand or profitability.
4. **Safety:** human review remains required for public release, spend, outreach, identity use, and strong claims.
