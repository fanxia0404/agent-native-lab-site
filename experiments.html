Experiment-first research

We test autonomous business loops before turning patterns into tools.

Agent Native Lab starts with concrete experiments: what an agent can observe, change, verify, measure, and improve; where a human still has to approve; and what failed along the way.

Can an AI agent operate and improve this public research site?

The first experiment uses this site as the test subject. The agent updates static pages, runs checks, publishes sanitized changes, reviews aggregate traffic, and records where owner approval is still required.

What every experiment should report

QuestionWhy it matters
What autonomous system are we testing?Keeps the scope concrete instead of writing generic AI commentary.
What can the agent observe?Defines the inputs the loop can actually use.
What can the agent change?Separates safe local action from risky public or irreversible action.
What must a human approve?Makes autonomy boundaries visible instead of hiding owner labor.
What did it cost and where did it fail?Turns experiments into operational evidence, not hype.
What pattern repeated?Only repeated patterns become checklists, playbooks, or tools later.

Tools come later

The evaluation worksheet is provisional. It should improve as experiments reveal repeated patterns. The site should not pretend the reusable framework is finished before enough autonomous-system tests exist.
