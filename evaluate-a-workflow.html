Practical worksheet

Score whether a workflow is ready for a safe AI-agent test.

Use this quick worksheet before building an agent. It helps decide whether to test the workflow with an agent, keep a human in the loop, or improve the process manually first.

How to use this page

Copy the checklist into a doc or spreadsheet. For one workflow, score each of the six dimensions and total the result out of 12.

0 = weak or unknown1 = partially true2 = strong and testable

What you get after 10 minutes

  • A 0–12 score for one workflow.
  • A decision: manual first, agent-assisted test, or bounded agent operation.
  • A first safe test that avoids sending, spending, publishing, or changing sensitive records.
  • A review gate and stop condition so the test does not drift into unsafe automation.

Where this fits

This is a pre-build triage worksheet for one candidate workflow. It is not an organizational AI-readiness assessment, not a production-readiness checklist, and not evidence that real users want the workflow automated.

Plain-English terms

Workflow the agent can help operate

A repeatable business task where an AI agent can do some work under clear limits, with humans approving risky steps.

Bounded agent operation

The agent can act only within named limits: specific inputs, specific allowed actions, review gates, non-sensitive records, and stop conditions.

Review gate

A point where a human must approve before the agent sends, spends, publishes, changes customer records, or makes a strong claim. Example: the agent may draft a reply, but a human must approve before it is sent.

The scoring checklist

Input clarityInputs are scattered, private, or undefined.Inputs exist but need cleanup or manual collection.Inputs are available, legal to use, and structured enough for repeat runs.

Interpret the score

0–4: Do not automate yet

Clarify the workflow manually. Find the input source, reduce risk, and define what success means before adding agents.

5–8: Agent-assisted test

Let the agent draft, research, classify, or recommend, but require human review before external action or strong claims.

9–12: Bounded agent operation

Run an internal-only or low-risk test where the agent can act inside clear limits, summarize non-sensitive results, and stop at review gates.

After scoring, define the first safe test

The score is not the decision by itself. The decision is the smallest test you can run without external harm.

FieldPrompt
Workflow unitWhat is one repeatable task the agent will handle?
Allowed inputsWhat approved sources can the agent read?
Allowed actionsWhat low-risk, reversible actions can the agent take without approval?
Review gateWhen must the agent stop and ask a human?
Success metricWhat metric should improve, and what baseline will you compare against?
First testWhat is the smallest internal or historical-data test you can run safely?
Stop conditionWhat failure means pause, revert, or redesign?

Red flags

Even a high score should be treated cautiously if the workflow includes:

  • regulated or high-stakes decisions;
  • irreversible spend or financial movement;
  • customer-visible actions without review;
  • unclear data rights or sensitive personal data;
  • no measurable output or feedback loop;
  • quality that cannot be reviewed quickly.

Important caveat

A high score means “worth testing under clear limits,” not “ready to run without oversight.” Public release, spend, outreach, identity use, and strong claims still need human approval.
