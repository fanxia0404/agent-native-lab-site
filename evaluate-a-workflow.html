Practical worksheet

Score whether an AI agent should run this workflow.

Use this quick worksheet before building an agent. It helps decide whether the workflow is ready for agent operation, needs human review, or should stay manual until the inputs and risks are clearer.

How to use this page

Copy the checklist into a doc or spreadsheet. Add one row for your workflow, score each dimension, and total the result out of 12.

0 = weak or unknown1 = partially true2 = strong and testable

Where this fits

This is a pre-build triage worksheet for one candidate workflow. It is not an organizational AI-readiness assessment, not a production-readiness checklist, and not evidence that real users want the workflow automated.

Plain-English terms

Agent-run workflow

A repeatable business task an AI agent can help operate under clear limits, not a claim that the agent can safely run the whole business alone.

Bounded agent operation

The agent may act only inside pre-approved limits and must stop at defined review gates.

Review gate

A point where a human must approve before the agent sends, spends, publishes, changes customer records, or makes a strong claim.

The scoring checklist

DimensionScore 0Score 1Score 2
Input clarityInputs are scattered, private, or undefined.Inputs exist but need cleanup or manual collection.Inputs are available, legal to use, and structured enough for repeat runs.
Action safetyWrong action can cause irreversible harm, spam, spend, or reputation damage.Agent can draft or recommend, but a human must approve action.Agent can act safely inside clear limits or on reversible local work.
ReviewabilityQuality is subjective or hard to inspect.A human can review quality, but it takes meaningful time.Output can be checked quickly by rules, tests, examples, or small review samples.
Measurable valueNo clear metric improves.Value is plausible but indirect.The workflow improves time saved, response speed, quality, cost, revenue, or another concrete measure.
Human timeRequires constant steering.Requires review at defined points.Can run mostly independently until a clear review gate appears.
Learning loopEach run is one-off and leaves no reusable improvement.Some notes or examples accumulate.Each run improves a checklist, dataset, template, process, content asset, or tool.

Interpret the score

0–4: Do not automate yet

Clarify the workflow manually. Find the input source, reduce risk, and define what success means before adding agents.

5–8: Agent-assisted test

Let the agent draft, research, classify, or recommend, but require human review before external action or strong claims.

9–12: Bounded agent operation

Run an internal-only or low-risk test where the agent can act inside clear limits, log results, and stop at review gates.

Red flags

Even a high score should be treated cautiously if the workflow includes:

  • regulated or high-stakes decisions;
  • irreversible spend or financial movement;
  • customer-visible actions without review;
  • unclear data rights or sensitive personal data;
  • no measurable output or feedback loop;
  • quality that cannot be reviewed quickly.

Important caveat

A high score means “worth testing under clear limits,” not “ready to run without oversight.” Public release, spend, outreach, identity use, and strong claims still need human approval.
