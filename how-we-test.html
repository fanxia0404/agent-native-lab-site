Can an AI agent help run the lab that studies agent-run work?

The lab tests one workflow at a time. For each workflow, it asks what the agent can do safely on its own, what evidence is produced, and where a human still needs to review the work.

Why this matters to a reader

The point is not the lab itself. The point is to make the test method visible: where the agent can continue safely, where it needs review, what it can measure, and what process mistakes repeat unless the rules are improved.

Hypothesis under test

A bounded workflow can be partly run by an AI agent when the agent has clear inputs, safe actions, measurable outputs, and explicit moments where a human must review or approve the work.

The workflow being tested

1. Observe

Read repo state, task records, source docs, and prior review findings instead of relying on chat memory.

2. Decide

Select the next allowed local task without asking for generic confirmation.

3. Act

Create or update local pages, docs, records, and checks.

4. Verify

Run static checks, link checks, and copy reviews before committing.

5. Measure

Record completed tasks, owner attention, cost, and repeated failure patterns.

6. Improve

Patch the process when failures reveal missing rules.
